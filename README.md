# 求人票抽出デモ（実務向けプロトタイプ）

ハローワーク系の求人票を想定し、**テキスト / PDF / URL** の3つの入力方法から、指定項目を見出し付きで1行ずつ整理して表示するフロントエンドデモです。

## 対応している入力方法

1. 求人票テキストを貼り付け
2. PDFファイルをアップロード（ブラウザでテキスト抽出できるPDFのみ）
3. 求人URLを入力（取得できない場合の案内付き）

## 使い方

1. `index.html` をブラウザで開く
2. 入力方法タブ（テキスト/PDF/URL）を選ぶ
3. データを入力して「抽出する」を押す
4. 抽出結果を確認し、「結果をコピーする」でそのままコピー
5. 必要なら「sample.txt を読み込む」でサンプルを入力欄へ読み込み

## 主な仕様

- 見出し語ベースの簡易抽出（厳密な自然言語解析は未実装）
- 未抽出項目は `詳しくは面談時にお問合せください。` を表示
- キャッチコピーは20〜120文字で仮要約
- 特長①〜③は20文字以内で仮生成
- 本文は150〜300文字で仮要約
- 給与は固定フォーマット（【月給】、【内訳】、昇給、賞与）で整形

## URL取得の制約

- 求人サイト側のCORS、bot対策、認証、robots制限などにより本文取得できない場合があります。
- 取得失敗時は画面に「本文を貼り付けてください」と案内を表示します。

## PDF対応の制約

- テキスト情報を持つPDFのみ抽出可能です。
- 画像PDF、スキャンPDF、保護PDFは抽出失敗することがあります。
- ブラウザ環境やネットワーク状況によって PDF ライブラリ読み込みに失敗する場合があります。

## sample.txt

- `sample.txt` に、指定項目を含むサンプル求人票を同梱しています。

## 今後のLLM連携案

1. 見出し抽出（ルールベース）→欠損項目のみLLM補完
2. 本文・特長・キャッチコピーをLLMで品質改善
3. 給与整形の妥当性チェック（単位・範囲・矛盾）をLLM + ルールで二重検証
4. URL先HTMLの重要箇所抽出をLLMで支援し、サイトごとの差異に追随


## 前処理（精度改善）

- PDF/URL入力では抽出前に `preprocessText(text, sourceType)` を実行しています。
- 主な処理:
  - `<br>` / `<br />` を改行へ変換
  - `Title:` / `ID:` / 投稿日 / 更新日 などのノイズ行を除去
  - URL由来のHTMLタグ断片（`<...>`）の除去
  - 連続空白・過剰改行の整理
  - 見出し候補（事業所名、職種、雇用形態、仕事内容、給与、勤務時間、休日、待遇、試用期間など）の前に改行挿入
- 画面のデバッグ表示で「生テキスト」と「前処理後テキスト」を比較確認できます。
